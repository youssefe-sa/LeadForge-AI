## Phase 2 : Découper `supabase-store.ts` (91KB → 6 fichiers)

**Le problème** : Ce fichier contient types, hooks React, LLM calls, Serper API, scoring, emails, images, CRUD leads, campaigns, templates, et config. Tout y est.

### Nouvelle structure :

```
src/lib/
├── types.ts           ← Lead, ApiConfig, EmailTemplate, ScheduledEmail (interfaces)
├── supabase.ts        ← Connexion Supabase (existant, inchangé)
├── hooks/
│   ├── useLeads.ts    ← Hook useLeads (CRUD + realtime)
│   ├── useApiConfig.ts← Hook useApiConfig + useEmailTemplates
│   └── useCampaigns.ts← Hook useCampaigns
├── llm/
│   ├── callLLM.ts     ← callLLM, callLLMForWebsite, retryWithBackoff
│   └── prompts.ts     ← generateWebsitePrompt, buildShortPrompt
├── api/
│   ├── serper.ts      ← serperFetch, searchLeadImages, deepSearchContact
│   ├── scoring.ts     ← calculateScore
│   └── images.ts      ← searchLeadImages, fetchImagesFromAPI
├── email/
│   ├── gmailSmtp.ts   ← Gmail SMTP sending
│   └── scheduled.ts   ← useScheduledEmails, cron logic
├── utils.ts           ← safeStr, safeNum, safeStrArr, mapColumns, exportLeadsCSV, validateEmailAdvanced, etc.
└── store.ts           ← Barrel export (re-export tout pour compatibilité)
```

### Principe :
- Chaque fichier expose des fonctions/hooks spécifiques
- `store.ts` re-exporte TOUT pour éviter de casser les imports existants
- Migration incrémentale : on crée les nouveaux fichiers, puis on update les imports un par un
- **Ne pas casser** les 12 fichiers qui importent de `supabase-store`
